# Klann linkage three.js viewer

Lightweight local visual validator for the Klann mechanism. Renders the
seven bodies (`torso`, `coupler`, `conn`, `b1`..`b4`) and sweeps them
through one full crank cycle so you can confirm by eye that the linkage
articulates correctly and the foot tip traces the expected Klann path.

## Design

Two-step pipeline:

1. **Bake** (`bake.py`): exports one STL per rigid body and a
   `frames.json` with a 4×4 world pose for each body at every frame
   (using `klann.create_klann_geometry` + 2D Kabsch alignment in
   `poses.py`).
2. **Render** (`index.html` + `main.js`): loads the meshes once,
   fetches `frames.json`, and sets `mesh.matrix` per animation frame.
   Three.js is imported from `esm.sh` via an importmap; no build step.

Re-run `bake.py` whenever you change `klann.py` constants or the body
topology. The browser just reloads.

## Run

```bash
just view          # bake + serve + open browser (recommended)
```

Or manually:

```bash
# 1. Bake the STL meshes + pose tables (once, or after linkage changes).
uv run python viewer/bake.py

# 2. Serve the viewer statically (anything that serves ./ works).
cd viewer && python -m http.server 8000
```

Open <http://localhost:8000> in any modern browser.

Flags on `bake.py`:

- `--out PATH` — output directory (default `viewer/data/`).
- `--frames N` — animation frame count (default 60).
- `--fps N` — playback rate the viewer should default to (default 30).

## UI

- **Slider** — scrub to any phase instantly.
- **Play / Pause** — RAF-driven playback at the baked FPS.
- **Red loop** — foot tip (`F`) trail over one cycle.
- **Orbit** — mouse drag to rotate; wheel to zoom; right-drag to pan.

## Files

```
viewer/
├── bake.py        # CLI: writes data/
├── poses.py       # 2D Kabsch + per-body SE(3) math
├── index.html     # importmap + DOM
├── main.js        # three.js scene + animation
└── data/          # gitignored — regenerated by bake.py
    ├── parts/*.stl
    └── frames.json
```

Tests: `tests/test_poses.py` covers the Kabsch primitive, torso
identity, 2-point round-trip, and the real Klann b1 link at a non-
reference phase.
